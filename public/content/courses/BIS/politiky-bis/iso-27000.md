# ISO/IEC 27000 — Information Security Management System

**ISO/IEC 27000** je *family standardů* pro Information Security Management System (ISMS). Internationally recognized, *certifiable*. Klíčový standard pro enterprise security governance.

## Rodina 27000

| Standard | Zaměření |
| :--- | :--- |
| **ISO 27000** | Overview + vocabulary |
| **ISO 27001** | *Requirements* pro ISMS (**certifiable**) |
| **ISO 27002** | *Controls* — implementační vodítka |
| **ISO 27003** | ISMS implementation guidance |
| **ISO 27004** | Monitoring + measurement |
| **ISO 27005** | Risk management |
| **ISO 27006** | Requirements pro auditing/certification bodies |
| **ISO 27017** | Cloud security |
| **ISO 27018** | Public cloud PII protection |
| **ISO 27701** | Privacy information management |

ISO 27001 + 27002 je *core duo*. 27001 říká *co*, 27002 *jak*.

## ISO 27001 — ISMS requirements

Standard *prescribes* high-level management system pro security. Není to *checklist* controls — je to *systematic process*.

### Clauses 4-10 (mandatory)

1. **Clause 4: Context of organization** — kdo jsme, kdo are stakeholders.
2. **Clause 5: Leadership** — top management commitment, security policy.
3. **Clause 6: Planning** — risk assessment, risk treatment plan.
4. **Clause 7: Support** — resources, competence, awareness, documentation.
5. **Clause 8: Operation** — implement risk treatment.
6. **Clause 9: Performance evaluation** — monitor, audit, management review.
7. **Clause 10: Improvement** — corrective actions, continual improvement.

### Annex A — controls

114 controls organized in 14 control categories (ISO 27001:2013) or 93 controls in 4 themes (ISO 27001:2022):

ISO 27001:2022 themes:

- **Organizational** controls (37).
- **People** controls (8).
- **Physical** controls (14).
- **Technological** controls (34).

Examples:

- A.5 Information security policies.
- A.6 Organization of information security.
- A.7 Human resource security (screening, training, termination).
- A.8 Asset management.
- A.9 Access control.
- A.10 Cryptography.
- A.11 Physical and environmental security.
- A.12 Operations security.
- A.13 Communications security.
- A.14 System acquisition, development, maintenance.
- A.15 Supplier relationships.
- A.16 Incident management.
- A.17 Business continuity.
- A.18 Compliance.

## ISMS — Information Security Management System

ISMS je *systematic approach* — celý organizační rámec, ne jen technical controls.

### Statement of Applicability (SoA)

Document listing *each* Annex A control, with *status*:

- Implemented (and reference to procedure).
- Not implemented (with *justification*).
- Compensating control (and reference).

SoA je *contract* — auditor kontroluje, že vše implemented as stated.

## Certifikace

Organization může *certifikovat* ISMS:

1. **Internal audit** — sami auditujeme.
2. **Pre-assessment** (optional) — vendor checks readiness.
3. **Stage 1 audit** — auditor reviews documentation.
4. **Stage 2 audit** — on-site, observes implementation.
5. **Certificate** issued, valid 3 years.
6. **Surveillance audits** — yearly during 3-year cycle.
7. **Recertification** — full audit every 3 years.

Audit dělá *certifikovaná* organizace (BSI, TÜV, DNV, atd.).

Cena: $30k-$300k podle velikosti organization + scope.

## Vztah k jiným standards

### ISO 27001 vs NIST CSF

| | ISO 27001 | NIST CSF ([[nist-csf]]) |
| :--- | :--- | :--- |
| Origin | global (ISO) | US Federal |
| Type | management system + controls | framework |
| Certification | yes | no (alignment only) |
| Focus | systematic mgmt | risk-based functions |
| Adoption | global enterprise | US federal, critical infra |

Many orgs use *both* — ISO 27001 for certifiable ISMS, NIST CSF for risk-based gap analysis.

### Mapping to other standards

ISO 27001 controls map to:

- NIST SP 800-53.
- PCI DSS.
- HIPAA.
- SOX.
- GDPR.

Mapping tables exist (e.g., NIST CSF informative references).

## Implementation challenges

- **Scope creep** — try to cover too much initially.
- **Documentation burden** — extensive paperwork.
- **Management buy-in** — without C-level support, fails.
- **Resource allocation** — security team often understaffed.
- **Continuous improvement** — common to be *one-shot* deployment, no real PDCA.

Successful ISMS:

1. Start small (focused scope).
2. Get management commitment.
3. Build security culture.
4. Iterate (PDCA).
5. Measure + improve.

## Real-world ISO 27001

Use cases:

- **Cloud providers** — AWS, GCP, Azure all 27001 certified.
- **Banking** — required by regulators.
- **Healthcare** — privacy compliance.
- **Government contractors** — RFP requirement.
- **B2B SaaS** — customer trust signal.

Often combined with:

- **SOC 2** (US audit framework, customer-facing).
- **PCI DSS** (payment cards).
- **HIPAA** (healthcare).
- **GDPR** (EU privacy).
