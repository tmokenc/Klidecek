# ICAO 9303 a struktura biometrického pasu

**ICAO Doc 9303** (Machine Readable Travel Documents, tedy strojově čitelné cestovní doklady) je *mezinárodní standard* pro elektronické pasy. Vydala jej **International Civil Aviation Organization** (ICAO), agentura OSN. Specifikuje fyzickou strukturu, datový formát, biometrické komponenty a bezpečnostní mechanismy. Od roku 2006 je elektronický pas *povinný* pro vstup do zemí, které se účastní amerického programu bezvízového styku *Visa Waiver Program*; dnes (2026) je prakticky univerzální po celém světě.

## Historie

* **1980** — ICAO vydává standardy MRTD pro klasické *papírové* pasy se strojově čitelnou zónou (MRZ, Machine Readable Zone).
* **1996** — návrh elektronických pasů (eMRTD).
* **2002** — finalizace bezpečnostních protokolů.
* **2004** — první státy začínají vydávat elektronické pasy.
* **2006** — elektronický pas se stává povinným pro Visa Waiver Program.
* **2021** — ICAO Doc 9303, 8. vydání — aktuálně platná verze.
* **2026** — elektronický pas vydávají prakticky všechny státy.

* **Knížka** se stránkami na razítka a víza.
* **Stránka s daty** obsahuje:
  * Fotografii a viditelné osobní údaje (bibliografické informace).
  * **MRZ** (Machine Readable Zone, strojově čitelná zóna) — 2 řádky textu čitelného přes OCR, umístěné dole.
* **RF čip** a **anténa** zalisované v *jedné* stránce nebo v *obálce*.
* **Typ čipu** — bezkontaktní podle ISO/IEC 14443 typu B (typicky), 13,56 MHz.

### Formát MRZ

```
P<CZEDOE<<JOHN<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<
12345678<8CZE9001011M2601010<<<<<<<<<<<<<<<<
```

* **Řádek 1:** P (passport, tedy pas) + kód země + jména.
* **Řádek 2:** číslo pasu + kontrolní číslice + národnost + datum narození + pohlaví + datum platnosti + osobní číslo + kontrolní číslice.
* Používá ho *pasová kontrola* (border control) přes OCR.
* Slouží jako zdroj pro odvození klíče **BAC** ([[bezpecnost-pasu]]).

## Logická datová struktura (Logical Data Structure, LDS)

ICAO 9303 specifikuje hierarchii datových souborů na čipu:

### Master File a Application Master File

* Kořenová úroveň a kořen aplikace elektronického pasu.

::: viz icao-lds-explorer "Strom Master File → DG1..DG16; klikněte na DG pro detail formátu, velikosti a přístupového protokolu."
:::

### EF.COM (Common Data Elements, společné datové prvky)

* Seznam přítomných souborů DG.
* Seznam hashů jednotlivých DG (pro pasivní autentizaci, Passive Authentication, [[bezpecnost-pasu]]).

### Datové skupiny (Data Groups, DG1 – DG16)

| DG | Obsah | Povinné? |
| :---: | :--- | :---: |
| **DG1** | Data z MRZ | ✓ |
| **DG2** | Obraz obličeje (JPEG 2000) | ✓ |
| DG3 | Snímky otisků prstů (1 nebo 2) | volitelné, v EU povinné |
| DG4 | Snímky duhovky | volitelné |
| DG5 | Zobrazený portrét (druhá fotografie obličeje) | volitelné |
| DG6 | (rezervováno) | — |
| DG7 | Zobrazený podpis / obvyklá značka | volitelné |
| DG8 | Zakódované rysy obličeje | volitelné |
| DG9 | Zakódované rysy otisku prstu | volitelné |
| DG10 | Zakódované rysy duhovky | volitelné |
| DG11 | Doplňkové osobní údaje | volitelné |
| DG12 | Doplňkové údaje o dokladu | volitelné |
| DG13 | Volitelné (národní data) | volitelné |
| DG14 | Bezpečnostní data (PACE, CA) | povinné při PACE/CA |
| DG15 | Veřejný klíč pro Active Authentication | volitelné, v EU povinné |
| DG16 | Osoby, které mají být informovány | volitelné |

### EF.SOD (Document Security Object, bezpečnostní objekt dokladu)

* Obsahuje **hashe** všech souborů DG.
* **Podepsaný** podpisovým certifikátem dokladu (Document Signing Certificate, DSC).
* Základ **pasivní autentizace** (Passive Authentication).

## Biometrické komponenty

### DG2 — obraz obličeje

* **Povinný** pro všechny elektronické pasy.
* Komprimovaný formátem **JPEG 2000**.
* **Rozlišení:** minimálně 300 dpi.
* **Velikost:** typicky ~15 KB.
* **Standard:** ICAO 9303 Part 9 a ISO/IEC 19794-5.

### DG3 — snímky otisků prstů

* **V EU povinné** od roku 2009 (Schengen, 2. generace).
* Typicky **2 otisky prstů** (levý a pravý ukazováček, případně oba ukazováčky).
* Komprimace **WSQ** (Wavelet Scalar Quantization).
* **Rozlišení:** 500 dpi.
* **Standard:** ICAO 9303 Part 9 a ISO/IEC 19794-4.

### DG4 — snímky duhovky

* **Volitelné** — používají je některé země.
* **JPEG 2000** nebo komprimované PNG.
* Snímky v blízkém infračerveném pásmu (NIR), ~70 KB.
* **Standard:** ISO/IEC 19794-6.

## Rozdíly mezi zeměmi

### EU (Schengen)

* **Obličej + 2 otisky prstů** jsou povinné.
* **AA + PACE + CA + TA** — všechny tyto protokoly jsou vyžadovány.
* **Brainpool křivky** pro asymetrickou kryptografii.
* **Silné EAC** (Extended Access Control) pro biometrická data.

### USA

* **Zpočátku jen obličej**; otisky prstů se uchovávají v samostatné imigrační databázi (US-VISIT).
* **PA** je vyžadována; **AA a EAC** vyžadovány *nejsou*.
* Jednodušší přístup k čipu.

### Česká republika

* **Obličej + otisky prstů** (podle nařízení EU).
* **AA** + PACE od roku 2009.
* **Vlastní CSCA** (Czech Country Signing CA, národní podpisová autorita).

### Japonsko, Korea, Austrálie, Kanada

* Obličej je povinný; otisky prstů se liší podle země.
* Vysoce kvalitní implementace čipu.

## Cíle biometrického pasu

1. **Silnější autentizace** — biometrika ověřuje *držitele*, ne jen toho, „kdo má pas".
2. **Prevence cestování s falešnými doklady.**
3. **Odhalování podvodů s identitou** (identity fraud).
4. **Snadnější pasová kontrola** — automatické brány eGates.
5. **Ochrana proti padělání** — čip je *velmi obtížné* napodobit.

## eGates — automatizovaná pasová kontrola

Moderní letiště mají **brány eGates** pro pasy EU:

1. **Cestující** vloží pas do skeneru.
2. **OCR** přečte MRZ.
3. **Čtení čipu** přes BAC/PACE — ověří PA + AA.
4. **Pořízení snímku** obličeje kamerou.
5. **Porovnání obličeje** s obrazem v DG2.
6. **Rozhodnutí** — brána se otevře, nebo je přivolán pracovník.

Propustnost: **~10–15 sekund na cestujícího** (oproti ~1 minutě při ruční kontrole).

Příklady:

* **Schengen ABC gates** (Automated Border Control, automatizovaná pasová kontrola).
* **UK ePassport gates.**
* **Global Entry** (USA — pro předem registrované cestující).
* **CANPASS, NEXUS** (Kanada).

## EES — Entry/Exit System (EU 2025)

[Entry/Exit System](https://travel-europe.europa.eu/ees_en) — biometrický systém pro cestující ze zemí mimo EU, kteří vstupují do Schengenu:

* **První vstup:** biometrická registrace (obličej + 4 otisky prstů).
* **Další přechody hranic:** automatické porovnání.
* **Sleduje překročení povolené doby pobytu** (visa overstays).
* **Zprovoznění:** postupné zavádění v roce 2025.

## ETIAS (2026)

[European Travel Information and Authorization System](https://travel-europe.europa.eu/etias_en):

* Cestovní povolení získané před cestou pro občany mimo EU, kteří nepotřebují vízum.
* Obdoba amerického ESTA.
* **Navázáno na data v čipu** elektronického pasu.

## Standardy

* **ICAO Doc 9303** Part 1–12 (nejnovější 8. vydání, 2021).
* **ISO/IEC 7501-1** — identifikační karty — strojově čitelné cestovní doklady ICAO.
* **ISO/IEC 14443** — bezkontaktní karty.
* **ISO/IEC 19794-5 (obličej), 19794-4 (otisk prstu), 19794-6 (duhovka)** — formáty biometrických dat.
* **BSI TR-03110** — technické specifikace PACE, CA, TA.
* **EAC v2.21** — Extended Access Control (rozšířené řízení přístupu).

## Soukromí (privacy)

* **Uchovávání biometrických dat** — čip data nevydává volně, ale stát je zná z okamžiku registrace (enrollment).
* **MRZ na dolním okraji** — teoreticky čitelná kýmkoli s NFC čtečkou (chrání ji ale BAC).
* **Sledování** — skryté čtení na dálku (nízké riziko, ale možné).
* **Sdílení dat mezi státy** — systém Prüm pro policii, ETIAS pro vízová rozhodnutí.

## Trendy

* **mDL (Mobile Driving Licenses)** — ISO/IEC 18013-5 — řidičské průkazy v telefonu.
* **EU Digital Identity Wallet** (EUDI) — univerzální mobilní identita.
* **Digitální cestovní doklady** (digital travel credentials) — výhledově nahradí fyzické pasy.
* **Sdílení biometrik mezi státy** — narůstá navzdory debatám o soukromí.
