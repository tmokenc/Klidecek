# ICAO 9303 a struktura biometrického pasu

**ICAO Doc 9303** (Machine Readable Travel Documents) je *mezinárodní standard* pro elektronické pasy. Vydán **International Civil Aviation Organization** (ICAO), agenturou OSN. Specifikuje fyzickou strukturu, datový formát, biometrické komponenty a bezpečnostní mechanismy. Od r. 2006 je e-pas *povinný* pro vstup do zemí pod *Visa Waiver Program* USA; dnes (2026) prakticky univerzální celosvětově.

## Historie

* **1980** — ICAO MRTD standardy pro klasické *paper* pasy s machine-readable zone (MRZ).
* **1996** — návrh elektronických pasů (eMRTD).
* **2002** — finalizace bezpečnostních protokolů.
* **2004** — první státy začínají vydávat e-pasy.
* **2006** — povinné pro Visa Waiver Program.
* **2021** — ICAO Doc 9303 8th edition — actual current version.
* **2026** — virtually all states issue ePassports.

* **Knížka** se stránkami pro stamps and visas.
* **Stránka s daty** obsahuje:
  * Photo + bibliographic info (visible).
  * **MRZ** (Machine Readable Zone) — 2 lines of OCR-readable text at bottom.
* **RF chip** + **antenna** embedded v *jedné* stránce nebo *obálce*.
* **Chip type** — bezkontaktní ISO/IEC 14443 typu B (typicky), 13.56 MHz.

### MRZ format

```
P<CZEDOE<<JOHN<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<
12345678<1CZE9001011M2601015<<<<<<<<<<<<<<<<
```

* **Line 1:** P (passport) + country code + names.
* **Line 2:** passport number + check digit + nationality + DOB + sex + expiry + personal number + check digits.
* Used by *border control* via OCR.
* Source pro **BAC** key derivation ([[bezpecnost-pasu]]).

## Logical Data Structure (LDS)

ICAO 9303 specifies hierarchy of data files on chip:

### Master File + Application Master File

* Root level + ePassport application root.

::: viz icao-lds-explorer "Strom Master File → DG1..DG16; klikněte na DG pro detail formátu, velikosti a access protokolu."
:::

### EF.COM (Common Data Elements)

* List of DG files present.
* Hash list of DGs (for Passive Authentication, [[bezpecnost-pasu]]).

### Data Groups (DG1 – DG16)

| DG | Content | Required? |
| :---: | :--- | :---: |
| **DG1** | MRZ data | ✓ |
| **DG2** | Face image (JPEG 2000) | ✓ |
| DG3 | Fingerprint images (1 or 2) | optional, EU mandatory |
| DG4 | Iris images | optional |
| DG5 | Displayed portrait (second face photo) | optional |
| DG6 | (reserved) | — |
| DG7 | Displayed signature / usual mark | optional |
| DG8 | Encoded face features | optional |
| DG9 | Encoded fingerprint features | optional |
| DG10 | Encoded iris features | optional |
| DG11 | Additional personal details | optional |
| DG12 | Additional document details | optional |
| DG13 | Optional (national data) | optional |
| DG14 | Security data (PACE, CA) | required if PACE/CA |
| DG15 | Active Authentication public key | optional, EU mandatory |
| DG16 | Persons to notify | optional |

### EF.SOD (Document Security Object)

* **Hashes** all DG files.
* **Signed** by Document Signing Certificate (DSC).
* Foundation of **Passive Authentication**.

## Biometrické komponenty

### DG2 — face image

* **Mandatory** for all e-pasy.
* **JPEG 2000** compressed.
* **Resolution:** 300 dpi minimum.
* **Size:** ~15 KB typical.
* **Standard:** ICAO 9303 Part 9 + ISO/IEC 19794-5.

### DG3 — fingerprint images

* **EU mandatory** since 2009 (Schengen 2nd gen).
* Typically **2 fingerprints** (left + right index, or both indexes).
* **WSQ** (Wavelet Scalar Quantization) compressed.
* **Resolution:** 500 dpi.
* **Standard:** ICAO 9303 Part 9 + ISO/IEC 19794-4.

### DG4 — iris images

* **Optional** — adopted by some countries.
* **JPEG 2000** or compressed PNG.
* **NIR** images, ~70 KB.
* **Standard:** ISO/IEC 19794-6.

## Country variation

### EU (Schengen)

* **Face + 2 fingerprints** mandatory.
* **AA + PACE + CA + TA** all required.
* **Brainpool curves** for asymmetric crypto.
* **Strong EAC** for biometric data.

### USA

* **Face only** initially; fingerprints retained via separate immigration database (US-VISIT).
* **PA** required; **AA, EAC** *not* required.
* Simpler chip access.

### Czech Republic

* **Face + fingerprints** (EU mandate).
* **AA** + PACE since 2009.
* **Vlastní CSCA** (Czech Country Signing CA).

### Japan, Korea, Australia, Canada

* Face mandatory; fingerprints variable.
* High-quality chip implementations.

## Cíle biometrického pasu

1. **Vyšší autentizace** — biometrika ověřuje *držitele*, ne jen "ten kdo má pas".
2. **Prevence cestování s falešnými dokumenty.**
3. **Detekce identity fraud.**
4. **Snadnější border control** — automated eGates.
5. **Anti-counterfeit** — chip je *velmi obtížné* napodobit.

## eGates — automated border control

Moderní letiště mají **eGates** pro EU pasy:

1. **Cestující** vloží pas do scanneru.
2. **OCR** přečte MRZ.
3. **Chip read** přes BAC/PACE — verifies PA + AA.
4. **Camera capture** of face.
5. **Face match** with DG2.
6. **Decision** — gate opens / referee called.

Throughput: **~10-15 sekund per cestujícího** (vs. manual ~1 minute).

Examples:

* **Schengen ABC gates** (Automated Border Control).
* **UK ePassport gates.**
* **Global Entry** (USA — pre-enrolled travelers).
* **CANPASS, NEXUS** (Canada).

## EES — Entry/Exit System (EU 2025)

[Entry/Exit System](https://travel-europe.europa.eu/ees_en) — biometric system for non-EU travelers entering Schengen:

* **First entry:** biometric enrollment (face + 4 fingerprints).
* **Subsequent crossings:** automated match.
* **Tracks overstays** (visa exceedances).
* **Operational:** rolling out 2025.

## ETIAS (2026)

[European Travel Information and Authorization System](https://travel-europe.europa.eu/etias_en):

* Pre-travel authorization for visa-exempt non-EU citizens.
* Like USA ESTA.
* **Tied to ePassport** chip data.

## Standardy

* **ICAO Doc 9303** Part 1–12 (latest 8th edition, 2021).
* **ISO/IEC 7501-1** — Identification cards — ICAO machine-readable travel documents.
* **ISO/IEC 14443** — bezkontaktní karty.
* **ISO/IEC 19794-5 (face), 19794-4 (fingerprint), 19794-6 (iris)** — biometric data formats.
* **BSI TR-03110** — PACE, CA, TA technical specifications.
* **EAC v2.21** — Extended Access Control.

## Privacy concerns

* **Biometric data retention** — chip nevypouští, ale stát ho zná po enrollment.
* **MRZ on bottom of chip** — readable by anyone with NFC reader (in theory; protected by BAC).
* **Tracking** — covert reading from distance (low risk, but possible).
* **Cross-border data sharing** — Prüm system for police; ETIAS for visa decisions.

## Trends

* **mDL (Mobile Driving Licenses)** — ISO/IEC 18013-5 — driving licenses on phones.
* **EU Digital Identity Wallet** (EUDI) — universal mobile identity.
* **Digital travel credentials** — replace physical passports eventually.
* **Cross-border biometric sharing** — increasing despite privacy debates.
