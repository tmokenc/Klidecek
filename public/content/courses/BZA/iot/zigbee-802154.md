# ZigBee a IEEE 802.15.4

**ZigBee** je nejstarší mass-market IoT mesh protokol — od r. 2003. Postavený na fyzické a linkové vrstvě **IEEE 802.15.4**, definuje **síťovou** vrstvu nad tím. Používá se v *smart home* (Philips Hue, IKEA Trådfri, Aqara, SmartThings), *industrial automation*, *smart meters*. Spravuje [Connectivity Standards Alliance](https://csa-iot.org/) (dříve ZigBee Alliance).

## IEEE 802.15.4 — fyzická + linková vrstva

* **Nelicencovaná ISM pásma:**
  * 1 kanál na 868 MHz (EU, 31,25 kb/s).
  * 16 kanálů na 2,4 GHz (globálně, 250 kb/s).
* **Topologie:** hvězda nebo P2P.
* **Typy zařízení:**
  * **FFD** (Full-Function Device) — plné schopnosti, může být koordinator nebo router.
  * **RFD** (Reduced-Function Device) — jen koncové zařízení, omezené resources.
* **Síť** vytváří **PAN koordinátor** (FFD), který přiděluje adresy.
* **Adresování:** 64-bit IEEE MAC + 16-bit short address (efektivnější pro packets v mesh).

### Bezpečnost v 802.15.4

* **Payload encryption** — definováno na linkové vrstvě.
* **Security level** — první 3 bity Security Control field:
  * `0` — žádné zabezpečení.
  * `1–3` — MAC pomocí **AES-CBC** (8/16 byte MAC).
  * `4` — šifrování **AES-CTR** (žádný MAC).
  * `5–7` — **AES-CCM** ([CBC-MAC] + CTR) — *standard* pro IoT, kombinace integrity + confidentiality.
* **Frame counter** — prevence replay, část CTR.
* **Key identifier** — identifikuje použitý klíč.

ZigBee přidává nad 802.15.4:

* **Topologie strom clusterů** — mesh s lehkou hierarchií.
* **Tři módy zařízení:**
  * **Koordinátor sítě** (FFD, PAN koordinator) — jeden v sítí, root celé tree.
  * **Router** (FFD) — *přeposílá zprávy* dál v mesh.
  * **Koncové zařízení** (FFD/RFD) — listy stromu, jen komunikuje s rodičem.
* **Stochasticky generované 16-bit NWK (short) adresy** — vygenerovány po připojení; koordinátor má vždy `0x0000`. Detekce kolizí zprávou `device_annc` s NWK i MAC adresou.

### Routing — AODV

ZigBee používá **Ad Hoc On Demand Distance Vector Routing (AODV)**:

* V případě potřeby cesty se vyhledá pomocí **Route Request (RREQ)** packets — broadcast přes mesh.
* Po obdržení **Route Response (RREP)** od cílové strany se *poskládají routing tabulky* na cestě.
* Bezpečnostní mód: RREQ/RREP může být *autentizován* síťovým klíčem.

## ZigBee bezpečnostní model

### Hierarchie klíčů

* **Master klíče** (Master Key) — zabezpečení výměny dalších klíčů. Default je *ZigBee 2007 Default TC Link Key* `5A 69 67 42 65 65 41 6C 6C 69 61 6E 63 65 30 39` (Pre-Configured Trust Center Link Key, *"ZigBeeAlliance09"* v ASCII).
* **Síťové klíče** (Network Key, NWK Key) — sdílené všemi zařízeními v síti. Broadcast komunikace.
* **Linkové klíče** (Link Key) — *per-pair* secret mezi dvěma zařízeními. Unicast komunikace.

### Dva bezpečnostní modely

#### Centralizovaný

* **Trust Center (TC)** — typicky koordinátor — spravuje všechny klíče.
* Nová zařízení dostávají NWK key přes Master Key.
* **Distribuce klíčů:** během *join* procedury TC dešifruje a předá NWK Key.
* **Joining new device** dostane defaultní TC Link Key z výroby — pokud uživatel nezmění, je *všem* znám.

#### Distribuovaný

* Žádný TC — koordinátor pouze řídí, ne shromažďuje klíče.
* Použito v *ZigBee 3.0* pro některé profile.
* Bezpečnost spíše na *zařízení-to-zařízení* (Smart Energy 1.2, Public Application Profiles).

### ZigBee 3.0 vs. ZigBee Pro vs. starší

* **ZigBee 2007** (legacy, ZigBee Pro) — standardní u většiny starších produktů.
* **ZigBee 3.0** (od 2015) — sjednocuje Smart Home + Light Link + dalších profiles. Vyšší bezpečnost (lepší klíče, mandatory unique install codes).

## Známé útoky

### Útok č. 1 — Default Trust Center Link Key

* **ZigBee 2007 default** TC Link Key je *veřejně známý*: `5A 69 67 42 65 65 41 6C 6C 69 61 6E 63 65 30 39`.
* Pokud zařízení připojené *bez* změny defaultního klíče (běžné u IKEA Trådfri starších gen.), útočník v dosahu může:
  1. Spustit *device discovery*.
  2. Sniffovat *Join procedure* s defaultním klíčem.
  3. Dešifrovat předanou *Network Key*.
  4. Připojit se *jako legitimní device* a injektovat příkazy.

::: viz zigbee-default-key "Krokuj join procedure: ZigBee 2007 (default TLK 'ZigBeeAlliance09' verejne znamy → utocnik dekoduje NWK Key) vs. ZigBee 3.0 (per-device install code → utok nemozny)."
:::

### Útok č. 2 — Touchlink Commissioning attack

[Cognosec 2017](https://cognosec.com/cve-2017-15361-zigbee-light-link-touch-link-vulnerability/):

* **ZigBee Light Link (ZLL)** Touchlink — automatický pairing přes proximity (zařízení se k sobě "přiblíží" a sjednotí).
* ZLL používá *master key* sdílený **všemi výrobci** — kompromitace = klonování *jakékoli* ZLL zařízení.
* Master key unikl v r. 2015 ([úniky reverz Philips Hue](https://lightaffliction.io/wp-content/uploads/2017/01/CVE-2017-15361-zll-touchlink.pdf)).
* Útočník v dosahu (až 100 m s vysílačem) může:
  * **Hijack** existing zařízení do své sítě.
  * **Reset** zařízení (mass reset všech Hue baz v okolí).
  * **Spustit firmware update** s podvrženou verzí (closed loop, ale možný).

### Útok č. 3 — Z-Shave (Z-Wave specific, podobný princip)

[Pen Test Partners 2018](https://www.pentestpartners.com/security-blog/z-shave-exploiting-z-wave-downgrade-to-compromise-esoteric-iot-devices/):

* Z-Wave (ne ZigBee, ale podobná rodina) — *downgrade* útok z S2 (secure) na S0 (legacy weak) security.
* Klíč S0 je odvozen z `0000...0000` (16 nulových bytes) jako *Network Key Establishment*.
* Útočník vynutí *re-pairing* s S0 → odhalí klíče.

ZigBee má analogii: vynucení *re-join* s defaultním Master Key → odhalí current Network Key.

### Útok č. 4 — Energy Depletion via control commands

* Smart bulb (Philips Hue) v sítí.
* Útočník (z legitimního ZigBee uzlu) vysílá *flicker* commands (toggle on/off) každou sekundu.
* Hue bulb spotřebuje *řádově víc* než pouhé čekání.
* Pro **battery-powered** ZigBee senzory (Aqara door sensors) je tento útok permanentně-DoS.

### Útok č. 5 — Wi-Fi-ZigBee interference (overlapping channels)

* ZigBee 2,4 GHz channels overlapnou s Wi-Fi.
* Útočník s vysokým-power Wi-Fi (např. spoofing flood) na ZigBee channel zruší legitimní komunikaci.
* Není útok *na* ZigBee, ale využívá fyziku sdíleného pásma.

### Útok č. 6 — Replay attack on frame counter rollover

Podobné LoRaWAN ([[lorawan]]):

* Frame counter v 802.15.4 je 32-bit; teoreticky bezpečný.
* Některé implementace ZigBee bez OTA update nikdy nezvládnou rollover správně — replay starých packetů projde.

### Útok č. 7 — KillerBee toolkit

[KillerBee](https://github.com/riverloopsec/killerbee) — open source ZigBee pen-test framework:

* **`zbid`** — discovery zařízení v dosahu.
* **`zbreplay`** — replay packetů.
* **`zbgoodfind`** — extrahovat NWK key z firmware dump.
* **`zbwireshark`** — Wireshark dissector pro ZigBee packets.

Vyžaduje *RZUSBSTICK* (Atmel) nebo *Apimote* — hardware dongles ~$50–200.

### Útok č. 8 — Skener vulnerable zařízení (BadZigBee)

[Cognosec/Bitdefender 2020](https://www.bitdefender.com/blog/labs/zerologon-uncovered) — komerční sken pro vulnerable Hue/IKEA/SmartThings zařízení v dosahu IoT systému (např. v rezidenčním komplexu, hotelu).

## Mitigace

* **Změna defaultního TC Link Key.** Pokud HW podporuje (ZigBee 3.0 install codes).
* **Install codes** (ZigBee 3.0) — per-device unikátní 8-byte kód, vytištěný na zařízení; uživatel ho zadá při pairingu pro **mutual authentication**.
* **Pravidelný update firmware** — gateway (např. Hue Bridge, SmartThings hub) musí dostávat security patches.
* **Vzdálenost** od veřejné cesty — fyzické umístění zařízení omezuje útočnou plochu.
* **Faradayova klec** — pro vysokocitové aplikace (industrial control, security IoT) izolovaná místnost s RF stínění.

## ZigBee vs. Z-Wave vs. Thread vs. Matter

| | ZigBee | Z-Wave | Thread | Matter |
| :--- | :---: | :---: | :---: | :---: |
| **Frekvence** | 2,4 GHz (also 868/915) | 868 MHz EU, 908 MHz US | 2,4 GHz | application layer, runs over IP |
| **Mesh** | ano | ano | ano | runs over IP networks |
| **Bezpečnost** | AES-CCM | AES-CCM (S2 mode) | AES-CCM, mDNS-IP | full stack, TLS-like over IP |
| **Open spec** | ano | částečně proprietary | open | open |
| **Smart home** | ano | ano | ano | ano (universal) |

**Matter (2022)** je nový standard od Apple+Google+Amazon+CSA, který se snaží sjednotit ekosystémy. Postaven na Thread/Wi-Fi/Ethernet, používá *Public Key cryptography* (ECDH, ECDSA, Curve25519) — bezpečnostně podstatně lepší než ZigBee.

Trend 2025: nové smart home produkty *opouštějí* ZigBee ve prospěch Matter; staré ZigBee zařízení zůstávají v provozu, ale s riziky.
